HTML и CSS можно менять на свое усмотрение

Задача:

Реализовать TODO-доску.

Функциональность:

Если при старте страницы hash (то что после #) пустой - выводить часть с классом bs-header. В противном случае выводить остальной блок.

Если хеш пустой - то при нажатии на кнопку "начать работу" сгенерировать случайный хеш (длина не важна) - и прописать его в location

Дальше мы будем называть этот хеш "именем доски".

Доска предоставляет следующие возможности:

- создание новой задачи (все 2 поля обязательны)

- перевод задачи в следующую колонку (в предыдущую нельзя по условиям задачи)

- удаление задачи кликом по креcтику

Важно: при открытии доски необходимо получить список текущих элементов на доске из localStorage под ключем, взятом из хэша (имя доски).
Хранить и получать данные вы будете в виде джейсон строки в localStorage:
[
  {
    "taskId":      "Task ID",
    "title":       "Task title",
	"description": "Task description",
	"status":      "todo | inprogress | done"
  },
  {
    "taskId":      "Task ID",
    "title":       "Task title",
	"description": "Task description",
	"status":      "todo | inprogress | done"
  }
]

Поля title, description - непустые строки, status - статус таска (один из трех вариантов: 'todo', 'inprogress', 'done').
Поле taskId генерируется при создании таска.
При любом изменении (перемещении, удалении или создании нового) таска - нужно обновлять содержимое localStorage под ключем, взятом из хэша (имя доски).
